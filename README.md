# randomCroquis / クロッキー練習ビューア

ローカルに保存した画像を一定時間ごとに切り替えながら表示し、クロッキー（短時間ドローイング）の練習を行うためのシンプルなビューアです。

GitHub Pages 版はこちらから利用できます：

- https://niwune2.github.io/randomCroquis/

---

## 機能概要

- ローカルの画像ファイル（複数）を読み込んでスライドショー表示
- 秒数指定による自動切り替え
- 前/次ボタンやキーボード操作による手動切り替え
- 画像のシャッフル表示（ON/OFF ＋ 再シャッフル）
- 全画面表示
- 終了時に任意の音声ファイル（例: VOICEVOX で生成した音声）を再生する機能
- 画像サイズが大きい場合は自動縮小して、はみ出さずに表示

## 使い方

### 1. 画像を読み込む

画面上部のボタンから、いずれかの方法で画像を読み込みます。

- **画像を選択**
  ローカルの画像ファイルを複数選択して読み込みます。

- **フォルダ**
  ブラウザが対応している場合（Chrome など）、フォルダを選択すると、
  そのフォルダ内の画像ファイルをまとめて読み込みます。

- **リセット**
  読み込んだ画像リストをクリアします。

> 対応画像形式は、一般的な `image/*`（PNG, JPEG, WebP など）です。

### 2. 再生・停止

中央のコントロールから操作します。

- **再生 / 停止**
  自動で一定間隔ごとに画像を切り替えます。

- **◀ / ▶**
  1枚ずつ手動で「前へ」「次へ」移動します（自動再生中でも使用可能）。

### 3. 秒数（切り替え間隔）

- 「秒数」入力欄に、表示時間（秒）を入力します。
- 5〜600秒の範囲で、5秒刻みで指定できます。
- 値を変更すると、次のカウントから新しい秒数が反映されます。

### 4. シャッフル表示

- **シャッフル: ON / OFF**
  ON のとき、画像の表示順がランダムになります。
  OFF のとき、読み込んだ順（または現在の並び順）のまま表示します。

- **再シャッフル**
  現在表示中の画像を先頭に固定したまま、残りの画像をランダムに並べ替えます。

### 5. 全画面表示

- **全画面** ボタンを押すと、ブラウザをフルスクリーン表示に切り替えます。
- もう一度 `Esc` キーを押すことで全画面表示を終了できます。

### 6. 終了音声

- **終了音声**
  手持ちの音声ファイル（`audio/*`）を1つ選択します。
  VOICEVOX などで生成した音声ファイルも利用できます。

- **音: ON / OFF**
  終了音を有効 / 無効にします。

- **鳴動モード**
  - 「毎回」：自動切り替えのたびに音声を再生します。
  - 「最後のみ」：最後の画像から最初に戻るタイミングでのみ音声を再生します。

- **音量**
  スライダーで 0〜1 の範囲で音量を調整します。

- **試聴**
  選択中の音声ファイルをその場で再生して確認します。

## ショートカットキー

テキスト入力欄などにフォーカスがない状態で有効です。

| キー          | 動作                         |
|---------------|------------------------------|
| `Space`       | 再生 / 停止                  |
| `→`           | 次の画像へ                   |
| `←`           | 前の画像へ                   |
| `F`           | 全画面表示の開始             |
| `S`           | シャッフル ON / OFF 切り替え |
| `+`           | 秒数を +5 秒                 |
| `-`           | 秒数を -5 秒                 |
| `R`           | 秒数を 60 秒にリセットし、シャッフル ON ＋再シャッフル |

## 画像表示の仕様

- 読み込んだ画像は縦横比を保ったまま表示します。
- 画像の長辺が大きすぎる場合は、自動的に縮小され、
  表示領域からはみ出さないように表示されます。
- 元画像が小さい場合は拡大せず、原寸のまま表示します。
- 背景は黒で、画像は中央に配置されます。

---

## 推奨環境

- デスクトップ版の最新ブラウザ（特に Google Chrome）を推奨しています。
- 「フォルダ」ボタンによるフォルダ読み込みは、
  `showDirectoryPicker` API に対応したブラウザでのみ利用できます。
- スマートフォンやタブレットでも動作しますが、
  キーボードショートカットなど一部機能は利用できない場合があります。
